Hierarchical Dependency Injection

Injector : RootInjector (providedIn: 'root')

  • Hierarchical Dependency Injection allows us to inject dependency at multiple levels and isolate sections of application and give them private dependency independent from the app
  • There are 5 types of injectors, EnvironmentInjector(root), ModuleInjector, ElementInjector, NullInjector, PlatformInjector
  • RootInjector creates single instance of dependency that is shared across the application.
  • ElementInjector creates new instance of dependency for every instance of component/element, defined by importing in providers array of component
  • ModuleInjector creates new instance dependency for each instance of the module that is shared between components declared in that module
  • DI looks for Dependency resolution from the same element it's defined in to upto top level, returns first encountered dependency.
  • NullInjector is at the top of component tree, if angular reaches at NullInjector while looking for dependency, it throws an error.
  • Hierarchical Dependency Injection helps in creating private isolated components/modules providing modularity
